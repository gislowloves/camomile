# 🌼 Chamomile Valley Docker. Lab1

## Цель лабораторной работы

Основная задача проекта — показать использование Docker файлов для:

- создания изолированной среды разработки
- автоматической сборки приложения
- воспроизводимого запуска программы
- упрощения развертывания проекта

## Инструменты

- GCC 13 (для сборки)
- C++17
- Ubuntu 24.04 (как минимальная среда для запуска)

## Структура проекта
```
chamomile/
│
├── romashkovaya_valley.cpp
├── Dockerfile
└── README.md
```
## Dockerfile

Контейнер собирается с использованием multi-stage сборки:

Этап сборки — компилируется программа на основе образа **gcc:13**.

Этап финального образа — используется лёгкий **ubuntu:24.04**, в него копируется готовый исполняемый файл и задаётся команда запуска.

## Сборка Docker образа

Находясь в директории проекта, выполните:
```bash
docker build -t chamomile .
```
Docker выполнит следующие действия:

- cоздаст промежуточный образ с компилятором для сборки приложения
- cкомпилирует программу
- cоздаст финальный образ на базе **Ubuntu** с уже готовым исполняемым файлом

## Запуск приложения

После успешной сборки образа приложение можно запустить командой:
```bash
docker run -it chamomile
```

## Преимущества использования Docker файлов

Использование контейнеризации позволяет:

- запускать приложение на любой системе
- избежать установки компилятора и зависимостей
- обеспечить одинаковую среду выполнения
- упростить переносимость проекта
